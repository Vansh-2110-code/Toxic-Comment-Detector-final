ENDPOINTS
=========

GET /
-----
Health check endpoint
Returns: API status and configuration

POST /api/predict
-----------------
Analyze single comment
Request: { "text": "comment" }
Returns: Predictions for all 6 categories

POST /api/batch-predict
------------------------
Analyze multiple comments
Request: { "texts": ["comment1", "comment2"] }
Returns: Array of predictions (max 100)

GET /api/stats
--------------
Get API statistics
Returns: Model info and categories


WORKFLOW
========

1. User visits frontend (index.html)
2. User enters comment text
3. Frontend sends POST to /api/predict
4. Backend preprocesses text
5. Backend vectorizes text (TF-IDF)
6. Model predicts toxicity levels
7. Backend returns JSON response
8. Frontend displays visual results
9. User can export results or analyze another


DEVELOPMENT WORKFLOW

Testing:

2. Run: python backend/test_api.py
